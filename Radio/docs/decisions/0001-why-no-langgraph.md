# ADR 0001：v1 不引入 LangGraph

**日期**：2026-05-15  
**状态**：采纳

## 背景

PRD 指定核心逻辑编排使用 LangGraph（状态机 + 循环判断）。

## 决策

v1 不引入 LangGraph，pipeline 用一条直线串行的 Python 函数实现。

## 理由

1. **v1 没有真正的状态机需求**：流程是 8 步直线（探测→录制→切片→STT→翻译→总结→推送→清理），没有循环、没有人工 HITL 分叉。
2. **LangGraph 学习成本不低**：引入它只为"看起来高级"，但带来一层抽象、调试更难、错误堆栈更深。
3. **v2 才有必要**：等 HITL 审批按钮（确认/重写/忽略）加入后，状态机价值才显现。届时再重构为 LangGraph 不晚。

## 后果

- 正面：代码量减少 ~30%，调试直接，新人 1 小时能看懂全链路。
- 负面：v2 加入审批流时需要重构 pipeline.py。可接受。
