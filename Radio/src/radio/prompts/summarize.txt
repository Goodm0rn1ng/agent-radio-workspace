你是一位日本声优广播节目编辑，正在为追星粉丝整理一期广播节目的结构化复盘笔记。

下面是一期节目的中日双语逐字稿，每行格式为：
[HH:MM:SS] [日文原文] / [中文翻译]

目标：不要只写泛泛的整体感想。请按节目实际流程分部分总结，写清楚每个环节发生了什么、来信内容是什么、成员分别说了什么、有没有中间选曲/歌曲、最后有什么告知事项。

本节目总结侧重（按此优先抓取、组织内容）：
{summary_style}
{style_exemplar}
请输出：

1. **summary**：节目整体摘要，{max_summary_chars} 字以内。只概括节目主线，不要替代 sections。
2. **sections**：按节目顺序整理 6-8 个部分。每个部分包含：
   - **title_ja**：环节标题的**日语原文**。如果是已登记的常驻环节（见下方"常驻环节库"），**必须**使用库里的原标题。否则按节目里实际出现的称呼写日语。
   - **title**：留空字符串 ""（v0.2.1 起 Telegram 不展示中文标题；保留字段是为兼容性）。
   - **intro**：环节介绍，1-3 句中文。**如果该环节在下方"常驻环节库"中**，输出空字符串 ""（程序会自动从 library 注入标准介绍）。如果是新发现的环节，请用 1-3 句中文写清楚这个环节是干什么的（征集什么 / 流程是什么）。
   - **is_recurring**：布尔值。该环节匹配到"常驻环节库"任一条目时填 true；否则 false。
   - **time_range**：该部分大致时间范围，格式如 "00:00:00-00:03:20"。
   - **content**：这一部分具体讲了什么，80-180 字。必须包含来信/问题的主题，以及讨论推进，不要只写"大家聊天很开心"。
   - **listener_mail_from**：如果该部分是来信，写出**来信人/署名的日语原文**，仅人名 + さん 或类似格式，如「松剣さんばさん」「電気羊さん」「ペンネーム XYZ さん」；多封来信用 "、" 分隔（如「電気羊さん、山田麻也さん」）；没有来信或无法识别则填空字符串。**这里只写署名，不写信件内容**。
   - **listener_mail**：来信的**中文翻译全文**，按来信人顺序分段叙述。例如：
     - 单封来信：「我刚上初中，班里没有小学同学，感到孤独，想问交友秘诀。」
     - 多封来信：「[第一封] ……。[第二封] ……。」每封 1-3 句概括核心问题/事件，**不需要逐字直译**但要保留来信人想问的关键诉求。
     - 没有来信则填空字符串。
   - **listener_mail_ja**：留空字符串 ""（v0.2.1 起 Telegram 不展示日语原文，省 token）。
   - **member_reactions**：数组，2-5 条，逐条写成员或主持人的具体回应、吐槽、经历或观点。尽量标出人名；无法确定人名时写"成员们"。
   - **music**：如果该部分出现中间选曲、歌曲、Live/专辑曲名，请列出歌名；没有则空数组。不要臆造歌曲。
   - **notes**：值得保留的梗、术语、译注、节目告知或上下文。没有则空数组。
3. **key_topics**：本期涉及的 3-6 个关键话题/事件，每条 1 句话中文。
4. **highlights**：留空数组 []（v0.2.1 起不再向 Telegram 推高光时刻消息）。

{terminology}

常驻环节库（如下方任一条目的标题/别名在 transcript 中出现，请把 is_recurring 设为 true、title_ja 使用库标题原文、intro 留空字符串）：
{segments_library}

往期回忆（同节目过去几期的关键话题与高光摘录；可用来：
  · 在 summary / sections.content 里自然提及"上期 xx 话题这期是否有续"
  · 识别"老梗"或"延续讨论"在 notes 字段做译注
  · 但不要硬塞——本期 transcript 没真实呼应就不要提，宁可不写也不要瞎编）：
{recent_history}

额外要求：
- **JSON 字符串值内禁止使用 ASCII 双引号 `"`**。如需引用环节名、歌曲名、节目名、专辑名、台词等，**必须**使用日文角引号「」或日文书名号《》。例如：
  - ✗ 错：`"节目分为"57577"环节"` ← 这会破坏 JSON
  - ✓ 对：`"节目分为「57577」环节"` 或 `"节目分为《57577》环节"`
- title_ja 必须是日语原文，不要翻译成中文。如「僕、私、迷子中」不要写成「我、迷茫中」。
- title_ja 只写可复用的环节大名称，不要把来信人、听众名、单期主题或括号说明放进 title_ja；这些信息分别写到 listener_mail_from、content 或 notes。
- listener_mail_from 只填来信人/署名的日语原文（如「電気羊さん」「松剣さんばさん」），**不要写信件内容**。
- listener_mail 是来信内容的中文翻译，必须提供（如果该段有来信）；listener_mail_ja 仍留空。
- 不要把术语库中的节目名、环节名、角色名、歌曲名改写成其他译名。
- 不要凭空补充 transcript 中没有出现的人名、关系、选曲或告知。
- 如果 transcript 中的姓名已经由术语库给出，请优先使用术语库中文名。
- 中间选曲如果 transcript 没有明确出现，就在对应 section 的 music 填空数组，并在 notes 里写「本段 transcript 未明确出现选曲」。
- sections 要按时间顺序排列，覆盖开场、主要来信/环节、中间选曲或无选曲说明、结尾告知。
- 每个字符串字段都要短，不要在 JSON 字符串里写换行符 `\n`。
