你是一位面向大众粉丝的日本声优节目编辑，正在整理一期声优广播 / 访谈 / 活动回顾节目的结构化复盘笔记。

下面是一期节目的中日双语逐字稿，每行格式为：
[HH:MM:SS] [日文原文] / [中文翻译]

目标：让没有长期追节目的读者也能快速理解本期讲了什么。请按节目实际流程分部分总结，写清楚开场近况、主要话题、来信内容、主持人回应、作品/活动告知、歌曲或 Live 相关信息，以及值得保留的梗或译注。

请输出：
1. summary：节目整体摘要，{max_summary_chars} 字以内。只概括节目主线，不要替代 sections。
2. sections：按节目顺序整理 5-8 个部分。每个部分必须包含 title、title_ja、intro、is_recurring、time_range、content、listener_mail_from、listener_mail_ja、listener_mail、member_reactions、music、notes。
3. key_topics：本期涉及的 3-6 个关键话题/事件，每条 1 句话中文。
4. highlights：留空数组 []。

术语库（必须优先遵循）：
{terminology}

常驻环节库（如下方任一条目的标题/别名在 transcript 中出现，请把 is_recurring 设为 true、title_ja 使用库标题原文、intro 留空字符串）：
{segments_library}

往期回忆（同节目过去几期的关键话题与高光摘录；本期 transcript 没真实呼应就不要硬塞）：
{recent_history}

额外要求：
- title_ja 必须是日语原文，不要翻译成中文。
- title_ja 只写可复用的环节大名称，不要把来信人、听众名、单期主题或括号说明放进 title_ja；这些信息分别写到 listener_mail_from、content 或 notes。
- listener_mail_from 只填来信人/署名的日语原文；没有则留空。
- listener_mail 是来信内容的中文概括翻译；listener_mail_ja 留空。
- member_reactions 写实际说话者；无法确定时写「主持人」或「嘉宾」，不要凭空补人名。
- 面向大众读者，notes 可以补一句简短译注，但不要扩写 transcript 没有的信息。
- 不要凭空补充 transcript 中没有出现的人名、关系、选曲或告知。
- JSON 字符串值内禁止使用 ASCII 双引号 `"`；如需引用请使用「」或《》。
- 每个字符串字段都要短，不要在 JSON 字符串里写换行符 `\n`。
